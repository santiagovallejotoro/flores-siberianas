## Key Decisions

- **No `_templates/` folder**: The feature page template is embedded directly in [[SCHEMA]]. Fewer files, same discipline. Claude reads SCHEMA before writing any feature page.
- **Graph-first linking**: Every feature, route, and concept is a `[[wikilink]]` even when the target page doesn't exist yet. Broken links appear in Obsidian Graph View and signal "this needs a page."
- **Three root files only** (SCHEMA / OVERVIEW / ARCHITECTURE). Everything else lives in `features/`, `logs/`, or `roadmap/`.
- **Colors documented with both HSL (exact, from code) and approximate hex** so they are usable in design tools as well as Tailwind classes.
- **Auth:** One Supabase pool; role in `public.profiles`; separate profile tables per role; two login URLs and two portal prefixes (see [[features/auth|auth]]).
